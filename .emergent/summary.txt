<original_problem_statement>
The user wants to build a mobile application named CheckMate. It's intended for apartment/home buyers to use as an interactive checklist during on-site property inspections.

**PRODUCT REQUIREMENTS:**
1.  **Interactive Configuration**: The user must be able to input the property's configuration (e.g., number of bedrooms, bathrooms, balconies).
2.  **Dynamic Checklist**: Based on the configuration, the app should generate a comprehensive checklist of items to inspect.
3.  **Item Inspection**: For each checklist item, the user must be able to:
    *   Mark its status (e.g., Pass, Fail, Needs Attention, Not Applicable).
    *   Add text notes or comments.
    *   Attach photos of any issues found.
4.  **Data Persistence**: Users should be able to save and resume inspections.
5.  **Report Generation**: After completing an inspection, the app must generate and allow exporting/sharing of the checklist report in both PDF and Excel formats.
6.  **Inspection Management**: Users should be able to manage multiple inspections, including deleting them.
</original_problem_statement>
